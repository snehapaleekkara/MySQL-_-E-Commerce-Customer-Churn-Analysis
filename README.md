E-Commerce Customer Churn Analysis (SQL)

This repository contains a specialized SQL database designed to analyze and predict Customer Churn within an e-commerce environment. The project focuses on identifying behavioral patterns, service satisfaction, and engagement metrics that influence whether a customer stays with a platform or leaves.

1. Customer Profiling & Segmentation
   
Demographics: Tracking tenure, city tier, and marital status to identify high-risk segments.

Preference Tracking: Analyzing preferred login devices (Mobile vs. Computer) and order categories (Electronics, Fashion, Grocery).

2. Behavioral Metrics
   
Engagement Levels: Measuring "Satisfaction Score," "Warehouse-to-Home Distance," and "Hour Spend on App" to gauge user friction.

Retention Indicators: Tracking the use of coupons, cashback amounts, and the interval since the last order to identify "at-risk" customers.

3. Issue & Resolution Tracking
   
Pain Points: A dedicated focus on customer complaints and their correlation with churn rates.

Technical Skills Demonstrated

Data Cleaning: Handling categorical data like Gender, MaritalStatus, and PreferredLoginDevice using ENUM and VARCHAR constraints.

Business Logic Implementation: Utilizing DECIMAL for financial precision (Cashback) and INT for tenure and satisfaction metrics.

Database Design: Crafting a schema that supports complex exploratory data analysis (EDA) for retail shopping behavior.
